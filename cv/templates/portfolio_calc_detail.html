Mini Calculator Project Report

This project is a Python-based mini calculator that performs basic arithmetic operations including exponentiation, division, multiplication, subtraction, and addition. The calculator processes a list of mathematical expressions and outputs the solution. It also includes validation to handle errors such as alphabetic characters in the equation and division by zero.

Modules and Functions:

  • Imports:
    • os: Used to restart the script in case of an error like division by zero.
    • sys: Provides access to command-line arguments and is used alongside os for restarting the script.
    • string.ascii_lowercase: Used to detect alphabetic characters in the input to prevent invalid inputs.
  • Key Variables:
    • operator: A list of operators supported by the calculator, including exponentiation (^), multiplication (*), division (/), addition (+), and subtraction (-).
  • Core Functions:
    • solve(equation): This is the main function responsible for solving the arithmetic expression. It checks for alphabetic characters in the equation, removes spaces, handles decimal points, and performs calculations in the correct order of operations (PEMDAS: Parentheses, Exponents, Multiplication/Division, Addition/Subtraction). The result is accumulated and returned at the end.
  • Support Functions:
    • spaces(values): Removes any spaces from the input values to ensure the equation is correctly processed.
    • full_stop(values): Detects decimal points and merges the values around it to form decimal numbers.
    • raised_to_pow(values): Handles exponentiation (i.e., raising numbers to a power) by solving expressions involving the ^ operator.
    • division(values): Solves division operations while checking for division by zero errors, which would otherwise cause runtime exceptions.
    • multiplication(values): Processes and solves multiplication problems.
    • subtraction(values): Merges subtraction operators and the numbers they apply to in the input list.
    • addition(values): Merges addition operators and the numbers they apply to in the input list.

Error Handling:

  • The program effectively handles:
    • Alphabetic Characters: The function solve checks if any alphabetic characters are present in the input equation, and if found, prompts the user with an error message to rewrite the equation.
    • Zero Division Error: In the division function, any attempt to divide by zero triggers a ZeroDivisionError. The program handles this gracefully by printing a relevant message and restarting the program.

Order of Operations:

The program follows the order of operations rule (PEMDAS) to ensure expressions are evaluated correctly:

  • Exponentiation (raised_to_pow).
  • Division (division).
  • Multiplication (multiplication).
  • Subtraction (subtraction).
  • Addition (addition).

Each of these operations is handled by individual functions that process the corresponding operator, updating the input list by replacing the operation with the result.

Project Features:

  • Basic Arithmetic Support: The calculator supports operations like exponentiation, division, multiplication, addition, and subtraction.
  • Decimal Handling: The program can process decimal numbers accurately.
  • Input Validation: Ensures that the equation does not contain any invalid characters (e.g., alphabetic letters) and handles division by zero.
  • Recursive Solution: The solve function recursively evaluates the expression, ensuring all operations are correctly processed.

Enhancements:

  • Parentheses Support: The current implementation does not handle expressions involving parentheses. Adding this feature would further improve the flexibility of the calculator.
  • User-Friendly Input: Implementing a more intuitive input mechanism, such as allowing users to input equations directly as strings, would make the calculator more user-friendly.
  • Improved UI: A simple graphical interface or even a command-line interface with better formatting could improve usability.
  • Additional Operations: Expanding the operations to include other mathematical functions like square root, logarithms, and trigonometric functions would enhance the calculator's capabilities.

Conclusion:

This mini calculator project successfully handles basic arithmetic operations, implements input validation, and ensures that errors like division by zero are gracefully handled. With further enhancements such as parentheses handling and improved user input, the calculator could become even more versatile.
